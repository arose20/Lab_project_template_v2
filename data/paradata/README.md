# Paradata

This directory contains information describing how data were generated and processed.

It captures computational and experimental context.

## Examples
- Pipeline parameters (e.g. CellRanger versions, flags)
- Run logs and execution metadata
- Software and environment versions
- QC thresholds used in processing

## Rules
- Must be automatically generated where possible
- Must be tied to specific pipeline runs or dataset versions
- Critical for reproducibility and auditing
